# bash-tester
Простой тестовый фреймворк на основе bash-скриптов

## Тестовый пример
Зайдите в папку example и запустите команду  
../bash-tester.sh .

## Описание теста
В разделе описаны правила для именования файлов и переменных в скриптах. Далее описан шаблон типового теста.

### Именование файлов
Для описания тестов используются bash-скрипты. Имя файла должно начинаться с префикса **test_** и иметь расширение **.sh**.
По этому шаблону имени фреймворк ищет тестовые скрипты и исполняет их. Поиск осуществляется в задаваемой пользователем папке и во вложенных папках.
Каждый тестовый скрипт запускается из своей директории (как текущей).  
В одном файле может определяться только один тест, не объединяйте несколько тестов в одном файле.  
Тестовый скрипт может использовать другие вспомогательные скрипты. Для именования вспомогательных скриптов не используйте префикс **test_**.  

### Переменные в скриптах
Фреймфорк использует свои внутренние переменные, они начинаются с префикса **BTEST_**. Пожалуйста, не создавайте свои переменные с таким префиксом.

### Шаблон теста
Для теста можно задать человекочитаемое имя в переменной BTEST_TEST_NAME, например:
BTEST_TEST_NAME="Single-function test"

Основная функция тестирования: **test**
Функции не передаются никакие аргументы. Однако вы можете задать пользовательские переменные в скрипте и использовать их в функцях.
